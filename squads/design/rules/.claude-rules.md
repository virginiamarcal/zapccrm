## Token Rules

- NEVER hardcode color, spacing, radius, shadow, or font values
- ALWAYS use CSS variables from the 3-tier token system:
  - **Primitive** (`--primitive-*`): Raw values, never use directly in components
  - **Semantic** (`--color-*`, `--spacing-*`): Purpose-based, preferred for component styling
  - **Component** (`--button-*`, `--card-*`): Component-specific overrides
- When unsure which token to use, query: `design_system.get_tokens { category: "color", tier: "semantic" }`

## Component Rules

- Before creating ANY new UI element, check existing components first
- Query: `design_system.suggest_component { use_case: "describe what you need" }`
- If a component exists, use it. If it doesn't, compose from existing primitives

## Accessibility Rules

- EVERY interactive element MUST be keyboard accessible
- EVERY form input MUST have an associated label (use `form-field`)
- EVERY icon-only button MUST have `aria-label`
- Color contrast MUST meet WCAG AA (4.5:1 normal text, 3:1 large text)
- Query a11y contract: `design_system.get_a11y_requirements { id: "button" }`
