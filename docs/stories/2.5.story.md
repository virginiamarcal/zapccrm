# Story 2.5: Pipeline Stage → WhatsApp Auto-Action

**Status:** Ready
**Sprint:** 4
**Epic:** Epic 2 - Integração WhatsApp↔CRM
**Story ID:** 2.5
**Complexity:** 8 pts

---

## Description

Como um **vendedor**,
Quero **que mover um deal para stage "Proposta Enviada" envie automaticamente uma mensagem WhatsApp**,
Para que **o cliente saiba que a proposta chegou**.

---

## Acceptance Criteria

- [x] **AC1:** Admin configura triggers em `/settings/automations`: Evento "Stage changed to [stage]", Ação "Send WhatsApp message", Seletor de template
- [x] **AC2:** Quando deal muda de stage: Função PL/pgSQL `on_deal_stage_change()` dispara, Lookup trigger configurado, Se existe: enqueue em QStash para envio
- [x] **AC3:** Mensagem sent com status tracking (sent/delivered/read)
- [x] **AC4:** Testes: move deal → mensagem enviada com 2s de latência

---

## Scope

### In Scope
- Automation trigger configuration (UI in Settings)
- PL/pgSQL trigger on deal stage change

- QStash integration for async send
- Message status tracking
- Tests for latency < 2s

### Out of Scope
- Template creation (Story 6.3 handles this)
- WhatsApp sending infrastructure (assumes Meta API available)

## Notes for Developer

- Automations table: org_id, event_type, event_value (stage_id), action_type, template_id, enabled
- Trigger on deals: UPDATE trigger that calls `on_deal_stage_change()`
- Template variables: {{contact_name}}, {{deal_name}}, {{stage_name}}, {{link}}
- QStash: use durable steps for retry (max 3 attempts)
- Status tracking: create message_status record in DB (sent/delivered/read)
- Latency: QStash should process in < 1s (user sends message in 2s total)
