**Status:** Ready
**Sprint:** 3

## Description

Como um **arquiteto**,
Quero **implementar o bridge layer com event-driven sync**,
Para que **dados fluam automaticamente entre WhatsApp e CRM sem polling**.

---

## Acceptance Criteria

- [x] **AC1:** Novo serviço `lib/bridge/sync-service.ts` com 3 componentes: `WhatsAppEventListener`, `CRMSyncEngine`, `BidirectionalTrigger`
- [x] **AC2:** Usa Supabase Realtime para subscribers em vez de polling
- [x] **AC3:** Testes unitários para cada componente
- [x] **AC4:** Logging detalhado (cada sync operação)
- [x] **AC5:** Error handling com retry (max 3, backoff exponencial)

## Notes for Developer

- Use TypeScript interfaces for component contracts
- Supabase Realtime: use `.on()` listeners (no polling)
- Retry backoff: 1s, 2s, 4s (exponential)
- All operations must log: timestamp, org_id, component, action, status
- Handle edge cases: duplicate events, out-of-order events
