## Notes for Developer

- Query activities + messages + deals in single query (use UNION in SQL)
- Ordering: `ORDER BY created_at DESC` (newest first)
- Filtering: filter by activity_type before rendering
- Pagination: use cursor-based pagination (created_at, id)
- Hover details: use Tooltip component (shadcn or Headless UI)
- Icons: use Lucide React for event types (mail, phone, check, etc)
- Avatar: fetch from users table (created_by field)
- Performance: limit to 20 per load (avoid huge lists)
