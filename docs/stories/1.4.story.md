# Story 1.4: Tenant Detection Middleware

**Status:** Ready
**Sprint:** 2
**Epic:** Epic 1 - Fundação Multi-Tenant
**Story ID:** 1.4
**Complexity:** 6 pts

---

## Description

Como um **usuário**,
Quero **acessar minha organização automaticamente ao logar**,
Para que **não precise selecionar manualmente**.

---

## Acceptance Criteria

- [ ] **AC1:** Middleware em `lib/middleware.ts` detecta `org_id` de: Header `X-Organization-ID` (API calls), Cookie `org_id` (web), Query param `?org={slug}` (URLs)
- [ ] **AC2:** Middleware valida que usuário é membro da org
- [ ] **AC3:** Redirection automática se org_id inválido
- [ ] **AC4:** Teste: API call sem org header retorna 400
- [ ] **AC5:** Teste: User acessando org diferente retorna 403

---

## Scope

### In Scope
- Implement Next.js middleware for tenant detection
- Support 3 detection methods: header, cookie, query param
- Validate user membership
- Auto-redirect invalid orgs
- Set context in request for downstream code

### Out of Scope
- Super admin impersonation (Story 1.5)
- Session management (Story 1.3 handles that)
- Role-based access control (Story 1.2 handles via RLS)

## Notes for Developer

- Middleware runs on EVERY request (keep it fast < 10ms)
- Priority: Header > Cookie > Query Param
- Cache tenant lookup (Redis, TTL 60s) to avoid DB hit per request
- Invalid org should redirect to org selection page (if user is member of multiple)
- API calls (X-Organization-ID header) should return 400, not redirect
