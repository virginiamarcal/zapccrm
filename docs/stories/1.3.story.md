# Story 1.3: Multi-Org Auth e Invite System

**Status:** InProgress
**Sprint:** 2
**Epic:** Epic 1 - Fundação Multi-Tenant
**Story ID:** 1.3
**Complexity:** 8 pts

---

## Description

Como um **owner**,
Quero **convidar membros do team via email**,
Para que **possam acessar os dados da organização com roles específicas**.

---

## Acceptance Criteria

- [ ] **AC1:** Endpoint `/api/auth/invite` cria token com expiracao 7 dias
- [ ] **AC2:** Email enviado com link `/join?token={token}`
- [ ] **AC3:** Page `/join` valida token e cria usuário + org_member com role (admin/member/viewer)
- [ ] **AC4:** Supabase Auth cria user, org_members cria relação
- [ ] **AC5:** Usuário logado vê dropdown de organizações (se member de múltiplas)
- [ ] **AC6:** Página `/settings/team` mostra membros, roles, opção de remover

---

## Scope

### In Scope
- Create invite endpoint with JWT token
- Email invite flow
- Accept invite & create org_member record
- Manage team members UI
- Role assignment (admin/member/viewer)

### Out of Scope
- Super admin panel (Story 1.5)
- Middleware tenant detection (Story 1.4)
- SSO / OAuth (future epic)

## Notes for Developer

- Use Supabase Auth for user creation (not manual DB insert)
- JWT token should include: invite_id, org_id, email (signed)
- Email must include plain text + HTML versions
- Prevent duplicate invites to same email
- Revoke expired tokens after 7 days
- Role defaults to 'member' if not specified
