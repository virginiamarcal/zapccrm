# Story 1.5: Super Admin Dashboard

**Status:** Ready
**Sprint:** 2
**Epic:** Epic 1 - Fundação Multi-Tenant
**Story ID:** 1.5
**Complexity:** 8 pts

---

## Description

Como um **super admin (Thales)**,
Quero **visualizar todas as organizações, usuários e métricas**,
Para que **possa debugar problemas e monitorar saúde do SaaS**.

---

## Acceptance Criteria

- [ ] **AC1:** Page `/admin/dashboard` mostra: Total de orgs, usuários, mensagens (último mês), Gráfico de atividade (msgs/dia), Lista de orgs com sorting por data, atividade
- [ ] **AC2:** Cada org clicável → dashboard interno (simulando usuário da org)
- [ ] **AC3:** Botão "Impersonate" para logar como membro de uma org (debugging)
- [ ] **AC4:** Audit log de todas as ações super admin
- [ ] **AC5:** Restrito a usuários com `is_super_admin = true`

---

## Scope

### In Scope
- Super admin dashboard page with metrics
- Organization listing with search/sort
- Impersonation feature (login as org member)
- Audit log for super admin actions
- Charts (orgs, users, message volume)

### Out of Scope
- Organization deletion (future security/compliance)
- Billing adjustments (Story 5.1 handles that)
- User management UI (org admins handle via Story 1.3)

## Notes for Developer

- Impersonation must log EVERY action to audit table
- Only super_admin users can access /admin/* routes
- Metrics should be cached (Redis, TTL 5min) to avoid DB overload
- Audit log: timestamp, super_admin_id, action, org_id, details
- Cannot impersonate another super admin (security)
- Session during impersonation is separate (logged out after)
